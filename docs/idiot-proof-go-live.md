# 小白保姆级上线指南

> 适合谁看：**老板、店长、运营负责人、完全不会写代码的人**
>
> 这份文档的目标只有一个：**让你按步骤把系统上线，并且真的能收钱、退款。**

---

## 先说最重要的事 ✅

这套系统上线，真正要做的就三件事：

1. **把微信支付账号准备好**
2. **把后台传到云端**
3. **自己用手机真的付 0.01 元，再真的退回来**

只要这三件事都做完，基本就到位了。

# 第一部分：微信支付账号怎么申请（合规收钱）💰

## 1. 为什么不能用一个账号收所有门店的钱？

说大白话：

- **谁的钱，就该进谁的商户账号。**
- 如果你用一个账号帮很多门店统一收钱，再自己私下分出去，这种做法很容易被微信判定为**二清**。
- **二清**你可以简单理解成：**钱先打到你这里，你再转给别人。**

风险是什么？

- 轻一点：风控、延迟结算、功能被限制
- 重一点：**商户号被冻结**
- 再严重一点：**小程序支付被停掉**

所以最稳妥的做法是：

- **每个门店自己申请自己的微信支付商户号**
- 如果你是平台帮很多店做系统，应该走**微信支付服务商模式**
- 如果你是同一个公司直营很多门店，应该认真评估**官方分账**

一句话记住：

> **不要图省事用一个账号代收所有门店的钱。**

---

## 2. 去哪里申请？

打开这个网站：

**https://pay.weixin.qq.com**

这是微信支付商户平台。

你进去之后，一般要做的是：

1. 注册商户
2. 提交资料
3. 等审核
4. 审核通过后，登录商户平台拿到后面要用的东西

---

## 3. 申请前要准备什么材料？

常见要准备这些：

- **营业执照**
- **法人身份证**
- **对公账户**，如果是公司
- **个体户银行卡**，如果是个体工商户
- **门店信息**
- **联系电话**
- **经营类目信息**

如果微信那边要求上传门头照、店内照片、经营证明，就按它页面提示准备。

不用背规则，记住一句话：

> **页面要什么，你就准备什么；资料尽量真实、完整、清晰。**

---

## 4. 通过以后，一定要拿到哪 3 样东西？

这 3 样东西非常重要。**没有它们，就没法正式收钱，也没法正式退款。**

### 第 1 样：商户号

你可以把它理解成：

> **微信支付给你这家店发的“收款账号编号”**

它一般是一串数字。

你以后要把它填到系统里。

名字你可能会看到这些：

- 商户号
- `MCH_ID`

你只要记住：

> **这是你家的微信收款编号。**

---

### 第 2 样：API v3 密钥

这个名字听起来很吓人，其实你把它理解成：

> **微信支付和你后台之间对暗号用的密码**

它主要用来做什么？

- 解密支付回调
- 让后台确认“这条支付结果是真的”

名字你可能会看到：

- API v3 密钥
- `API_V3_KEY`

你拿到之后：

- **不要发微信群**
- **不要发给不相关的人**
- **不要写进 Word、Excel 到处传**

---

### 第 3 样：API 证书 / 私钥文件

这个是最容易被忽略，但又**最不能丢**的东西。

你可以把它理解成：

> **退款专用钥匙**

尤其是这个文件：

- `apiclient_key.pem`

请把它理解成：

> **退款钥匙文件**

它非常重要，因为：

- **退款要用**
- 分账也可能要用
- 有些高级支付能力也会用到

### ⚠️ 重点提醒

这个文件下载后，请立刻做这几件事：

1. **保存在电脑一个安全的位置**
2. **再备份一份到公司安全网盘或 U 盘**
3. 文件名不要乱改
4. 不要随便发给别人
5. 不要丢进群里

你可以专门建一个文件夹，例如：

```text
微信支付资料-请勿外传
```

里面放：

- 商户号记录
- API v3 密钥记录
- 证书序列号
- `apiclient_key.pem`

一句话记住：

> **这个私钥文件要是丢了，退款会很麻烦。**

---

## 5. 这 3 样东西在哪里找？

大白话版：

- **商户号**：登录微信商户平台后，首页或账户信息里能看到
- **API v3 密钥**：商户平台里找“API 安全”
- **API 证书 / 私钥文件**：商户平台里申请 API 证书时下载

你可能还会看到一个东西叫：

- **证书序列号**
- `CERT_SERIAL_NO`

你把它理解成：

> **这张证书的编号**

它也要一起保存好。

# 第二部分：怎么把代码放上云端（一键部署）☁️

这一部分你不用懂什么叫云函数，也不用懂什么叫数据库原理。

你只需要理解：

- **第一条命令** = 把后台传到云端
- **第二条命令** = 把数据库和超级管理员准备好

---

## 1. 先打开终端（命令行）

如果你是 Windows：

### 方法 1：最简单

1. 打开项目文件夹
2. 在空白处点击鼠标右键
3. 看看有没有：
   - **在终端中打开**
   - **Open in Terminal**
   - **在 PowerShell 中打开**
4. 点它

### 方法 2：不会的话

1. 按键盘 `Win`
2. 输入：`cmd`
3. 或输入：`PowerShell`
4. 打开后，再切到项目文件夹

如果有人已经帮你把项目放到电脑上，最简单的办法就是：

> **让技术同事先帮你把终端打开到项目目录。**

---

## 2. 第一步：把后台传到云端

在终端里输入这句：

```bash
npm run deploy:cloud
```

然后按回车。

你把它理解成：

> **“把后台上传到腾讯云开发”**

它会做什么？

- 把系统后面的几个后台服务传上去
- 让小程序和后台有地方可以运行

你要做的只有两件事：

1. 输入命令
2. 等它跑完

如果屏幕上一直在滚动英文，不要慌，这是正常的。

---

## 3. 第二步：初始化数据库和超级管理员账号

在终端里再输入这句：

```bash
npm run db:init
```

然后按回车。

你把它理解成：

> **“把系统第一次要用的数据准备好”**

它会做什么？

- 创建需要的数据表
- 初始化系统数据
- 初始化超级管理员账号

如果前面支付和环境资料都准备好了，这一步会很省事。

---

## 4. 微信支付那 3 样东西，怎么填进腾讯云开发？

这一步是很多人最怕的，其实你只要按着点就行。

### 先记住一句话

- **商户号**可以记到系统配置里
- **密钥和私钥文件**要放到 `payApi` 的环境变量里

你不用理解“环境变量”是什么。

你把它理解成：

> **后台偷偷保存的重要密码柜**

---

## 5. 在 CloudBase 控制台里怎么点？

### 第一步：打开腾讯云开发控制台

打开：

**https://tcb.cloud.tencent.com**

然后：

1. 登录你的账号
2. 进入你这个项目对应的环境

---

### 第二步：找到云函数

进去后，找页面左边菜单。

一般会看到类似：

- 云函数
- Cloud Functions

点进去。

---

### 第三步：找到 `payApi`

在云函数列表里，找到名字叫：

**`payApi`**

点进去。

它就是负责支付和退款的地方。

---

### 第四步：找到“环境变量”

进入 `payApi` 后，找页面上的按钮或标签。

通常会看到类似：

- 环境变量
- 环境配置
- Variables

点进去。

---

### 第五步：把支付资料填进去

把你刚才拿到的东西填进去。

常见会填这些名字：

```text
API_V3_KEY
CERT_SERIAL_NO
PRIVATE_KEY
```

如果技术同事还让你填：

```text
PAY_CALLBACK_SECRET
```

也一起填。

### 大白话解释一下

- `API_V3_KEY`：微信和后台对暗号的密码
- `CERT_SERIAL_NO`：证书编号
- `PRIVATE_KEY`：退款钥匙

### 特别注意

`PRIVATE_KEY` 一般不是一句短文字，它可能是一大段内容，看起来像这样：

```text
-----BEGIN PRIVATE KEY-----
这里是一大段内容
-----END PRIVATE KEY-----
```

**复制的时候要完整复制，不要漏掉头尾。**

---

## 6. 商户号填在哪里？

商户号一般对应系统里的支付配置。

你可以理解成：

> **告诉系统：这家店是用哪个微信支付商户号收钱**

如果你是让技术同事代填，他们会把它填进系统的支付配置里。

如果你要自己在后台里填，记住填的是：

- **商户号**
- 也就是 `MCH_ID`

# 第三部分：一分钱真实支付与退款测试（上线前最后一哆嗦）🧪

这一步非常关键。

为什么？

因为：

- **代码测试通过，不代表真的能收钱**
- **真的收了钱，也不代表真的能退款**

所以一定要自己真测一次。

---

## 1. 先在微信开发者工具里点“预览”

操作步骤：

1. 打开微信开发者工具
2. 打开这个小程序项目
3. 确认已经选对小程序 AppID 和云环境
4. 看顶部按钮，找到：
   - **预览**
5. 点它
6. 屏幕会弹出一个二维码
7. 用你自己的手机微信扫码

你手机里就能打开预览版小程序。

---

## 2. 准备一个 0.01 元测试商品

最简单的方法有两个：

### 方法 A：把一个商品临时改成 0.01 元

### 方法 B：专门新建一个测试商品

价格写成：

**0.01 元**

这样就算你真付一次，也几乎没成本。

### 注意

测完以后，**一定记得改回正常价格**，或者把测试商品下架。

---

## 3. 真付 0.01 元

手机扫码进入预览版小程序后：

1. 找到那个 0.01 元商品
2. 正常下单
3. 正常付款
4. 用你自己的微信把 0.01 元付出去

然后看 3 件事：

### 第一件事：微信有没有真的扣款？

如果微信支付成功，你会看到支付成功提示。

### 第二件事：小程序订单有没有变成“已支付”？

去小程序订单列表里看。

你要看到订单状态变成：

- **已支付**
- 或类似“支付成功”

### 第三件事：后台有没有看到这笔订单？

如果你有后台，就去后台订单列表里看一下。

如果后台也能看到，说明支付链路基本打通了。

---

## 4. 真做一次退款

现在开始做退款测试。

你可以在：

- 小程序里点退款
- 或管理后台里点退款审核

按你们现在实际上线准备的流程操作。

然后继续看 3 件事：

### 第一件事：微信有没有退款通知？

最直接的理解就是：

> **你那 0.01 元有没有退回到微信里**

如果钱真的退回来，说明退款动作成功了。

### 第二件事：系统里的订单有没有变成“已退款”？

你要去订单页面看状态。

理想情况是：

1. 先变成：**退款中**
2. 然后再变成：**已退款**

你也可能看到类似这些字：

- `refunding`
- `refunded`

不用怕，这两个就是：

- `refunding` = 退款中
- `refunded` = 已退款

### 第三件事：后台退款记录有没有同步？

如果后台也能看到退款状态，说明整条退款链路是通的。

---

## 5. 什么情况才算真的可以上线？

下面这 5 件事，缺一不可：

- ✅ 微信支付商户号已经申请好
- ✅ API v3 密钥已经配好
- ✅ API 证书和私钥文件已经下载并安全保存
- ✅ 真实 0.01 元支付成功
- ✅ 真实退款成功

只要这 5 个都完成，基本就可以放心很多。
