# “浴小主母婴知识”隐私指引提审修正说明

## 结论

本次审核驳回的核心问题不是接口本身不能用，而是**隐私保护指引里填写的“接口用途说明”和当前小程序真实场景不匹配**。

结合当前仓库代码与微信官方当前文档，建议按下面方式调整微信公众平台里的《用户隐私保护指引》：

| 接口 | 当前代码是否真实使用 | 建议 |
| --- | --- | --- |
| 收集你的昵称、头像接口 | 是 | 保留，并按“完善个人资料并展示头像昵称”填写说明 |
| 访问你的摄像头接口 | 是 | 保留，并改成“宝宝日常拍照记录”相关说明 |
| 收集你的手机号接口 | 是 | 保留，并改成“微信授权手机号登录/身份校验”相关说明 |
| 收集你选中的照片或视频信息接口 | 是 | 保留，并改成“用户主动选择照片并保存记录”相关说明 |

## 代码核对结果

### 1. 昵称、头像

当前版本已接入微信官方现行的头像昵称填写能力：

- `miniapp/pages/profile/profile.wxml`
- `miniapp/pages/profile/profile.js`
- `miniapp/cloudfunctions/opsApi/index.js`

目前的真实业务链路是：

- 用户在个人中心主动选择头像
- 用户在个人中心主动填写昵称
- 小程序保存头像昵称并展示在个人中心
- 邀请记录、门店后台客户列表也会使用该头像昵称

根据微信官方当前文档，“收集你的昵称、头像”对应的是：

- `<button open-type="chooseAvatar">`
- `<input type="nickname">`

同时，官方文档已明确说明：

- `wx.getUserProfile` 已回收
- `wx.getUserInfo` 已回收
- `open-data` 的 `userNickName` / `userAvatarUrl` 不再返回真实值

因此，这条接口**可以保留**，但用途说明必须围绕当前真实场景来写，不能再写成宽泛的“完善资料”“优化服务”“个性化推荐”。

### 2. 手机号

手机号授权登录在两个页面有真实使用：

- 个人中心登录：`miniapp/pages/profile/profile.js`
- 宝宝日常页登录弹窗：`miniapp/pages/tongue/tongue.js`

服务端会通过微信开放接口换取手机号并完成登录态绑定：

- `miniapp/cloudfunctions/opsApi/index.js`

这条用途应当围绕：

- 微信授权手机号登录
- 会员身份校验
- 查询订单、套餐、成长记录等登录后内容

不要写成泛泛的“完善资料”“优化服务”“营销触达”。

### 3. 摄像头

“宝宝日常”页确实允许用户直接拍照：

- `miniapp/pages/tongue/tongue.js`

这里调用的是：

- `wx.chooseMedia`
- `sourceType: ['album', 'camera']`
- `mediaType: ['image']`

所以摄像头用途应明确写成：

- 用户主动拍摄照片
- 用于保存本次记录

不要扩展到与当前审核态无关的用途。

### 4. 相册/已选照片信息

同一个 `wx.chooseMedia` 调用也会读取用户主动选择的图片文件信息，用于上传和保存记录：

- `miniapp/pages/tongue/tongue.js`

注意当前代码里 `mediaType: ['image']`，也就是**当前实际只处理图片，不处理用户主动选择的视频**。  
因此描述时可以沿用微信的标准接口名，但说明里应聚焦“照片”。

## 建议填写文案

以下文案可以直接用于微信公众平台后台填写。

### 收集你的昵称、头像接口

建议填写：

`用于用户在个人中心主动选择头像、填写昵称，完善个人资料，并在个人中心、邀请记录及门店后台客户列表中展示用户头像昵称。`

### 访问你的摄像头接口

建议填写：

`用于用户在“宝宝日常”页面主动拍摄宝宝照片，以便生成并保存本次成长记录。`

### 收集你的手机号接口

建议填写：

`用于用户通过微信授权手机号完成登录与身份校验，以便查询订单、套餐、成长记录等仅登录后可使用的内容。`

### 收集你选中的照片或视频信息接口

建议填写：

`用于读取用户主动选择或拍摄的照片文件信息，并上传保存成长记录。当前版本仅处理图片。`

## 填写时的注意事项

1. 不要写“优化体验”“精准营销”“个性化推荐”这类泛化表述。
2. 不要把多个不相关目的塞进同一条说明。
3. 相机、相册类接口要明确写“用户主动拍摄/主动选择”。
4. 手机号接口要明确写“登录/身份校验”，不要写成普通资料收集。
5. 昵称头像接口要明确写“个人中心完善资料并展示”，不要写成笼统的“完善信息”。
6. 当前审核态页面文案是“宝宝日常”，不要把“AI 看舌象”“体质诊断”作为本次审核说明主描述。

## 重新提审前检查清单

1. 在微信公众平台保留“收集你的昵称、头像接口”，并改成上面的个人资料展示文案。
2. 把手机号、摄像头、相册三条说明改成上面的精简版本。
3. 如果后台还有补充文件，文案也保持一致，不要出现“AI 诊断、营销、无关资料收集”等扩大用途描述。
4. 真机确认个人中心能正常选择头像、输入昵称并保存成功。
5. 重新生成并核对《用户隐私保护指引》预览页，确认公开展示内容与当前页面功能一致后再提交审核。
